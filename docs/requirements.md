# GPT Image Editor - 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
GPT Image Editor

### 1.2 目的
OpenAI の GPT Image 2 (gpt-image-1) の Edit エンドポイントを活用し、ユーザーがアップロードした画像をテキスト指示で編集できる Web アプリケーションを構築する。

### 1.3 技術スタック
| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |
| UIコンポーネント | shadcn/ui |
| API | OpenAI Images Edit API (`gpt-image-1`) |
| ランタイム | Node.js |

---

## 2. 画面構成

UIは3カラムレイアウトで構成する。

### 2.1 ヘッダー
- アプリ名「GPT Image Editor」とBetaバッジの表示
- ダークモード切替トグル

### 2.2 左サイドバー（入力パネル）

#### 2.2.1 画像アップロード
- ドラッグ&ドロップ対応のアップロードエリア
- クリックでファイル選択ダイアログを開く
- 対応フォーマット: PNG, JPEG, WebP
- アップロード後にサムネイルプレビューを表示

#### 2.2.2 編集指示入力
- テキストエリアで編集内容を自然言語で入力
- プレースホルダーでガイドを表示（例：「空を夕焼けに変更してください」）

#### 2.2.3 おすすめプロンプト
- プリセットのプロンプトボタンを複数表示
- クリックでテキストエリアに反映

#### 2.2.4 モデルと設定
- モデル選択ドロップダウン（`gpt-image-1`）
- 画像サイズ選択（`1024x1024`, `1536x1024`, `1024x1536` など）

#### 2.2.5 編集実行ボタン
- 「編集を実行」ボタン（紫色のプライマリボタン）
- 処理中はローディングインジケーターを表示

### 2.3 メインエリア（中央）

#### 2.3.1 編集プレビュー
- 元画像 / 編集後画像の表示
- 編集前後の比較が可能な表示

#### 2.3.2 編集履歴
- 過去の編集結果をサムネイル一覧で表示
- サムネイルクリックでプレビューに反映

### 2.4 右サイドバー
- プロンプト履歴の表示

## 3. 機能要件

### 3.1 画像アップロード機能
| 項目 | 内容 |
|------|------|
| 対応形式 | PNG, JPEG, WebP |
| 最大ファイルサイズ | 25MB（API制限に準拠） |
| バリデーション | ファイル形式・サイズのチェック、エラーメッセージ表示 |

### 3.2 画像編集機能
| 項目 | 内容 |
|------|------|
| API | OpenAI Images Edit API |
| モデル | `gpt-image-1` |
| 入力 | アップロード画像 + テキストプロンプト |
| 出力 | 編集後の画像（Base64 or URL） |

### 3.3 編集履歴機能
- セッション内で編集結果を保持
- 履歴からの再表示・ダウンロード

### 3.4 画像ダウンロード機能
- 編集後の画像をローカルにダウンロード
- 「ダウンロード」ボタンを提供

### 3.5 ダークモード
- ライト/ダーク切替対応
- システム設定に連動（初期値）

## 4. API設計

### 4.1 画像編集 API
```
POST /api/edit
Content-Type: multipart/form-data

リクエスト:
  - image: File (必須) - 編集対象の画像
  - prompt: string (必須) - 編集指示テキスト
  - size: string (任意) - 出力画像サイズ

レスポンス:
  {
    "imageUrl": string,  // 編集後画像のURL or Base64データ
    "revisedPrompt": string  // API が修正したプロンプト
  }
```

---

## 5. 非機能要件

### 5.1 パフォーマンス
- 画像アップロードは即座にプレビュー表示
- API応答待ち中はスケルトン/ローディング表示

### 5.2 セキュリティ
- OpenAI APIキーはサーバーサイドのみで使用（環境変数）
- クライアントにAPIキーを露出させない

### 5.3 レスポンシブ
- デスクトップ表示を主対象とする
- 最低限のモバイル対応（カラム折りたたみ）
